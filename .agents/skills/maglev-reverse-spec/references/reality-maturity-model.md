# Reality Maturity Model

## RL-1: Sketchy
- 只有高层描述
- 缺少验收标准
- 缺少架构与流程

## RL-2: Structural
- 已有入口、核心 API、主要数据模型
- 已能描述核心字段、关系和跨层映射
- 已能描述模块拓扑
- 仍不足以支撑直接开发

## RL-3: Ready to Engineer
- 具备组件划分
- 具备关键时序或主流程图
- 具备核心协议与状态变化
- 具备数据字典、字段约束和关键结构生命周期
- 具备可测试 AC
- 完成关键鲁棒性检查

## RL-4: Verifiable
- 代码双向追溯齐备
- 测试覆盖路径明确
- 具备验证报告或等价质量门禁
