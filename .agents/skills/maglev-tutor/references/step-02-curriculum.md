# Step 2: Adaptive Curriculum (自适应课程)

## 目标
基于 Profiling 结果，推送差异化内容。
(Mode: **Deep**)

## 课程逻辑

### 1. User Track (使用者路径)
*   **Target**: 想要高效完成业务迭代的开发/产品/测试。
*   **Modules**:
    *   [U1] **Maglev Flow**: 现状 -> 需求 -> 方案 -> 实施 -> 验证（更贴近传统协作流的理解方式）。
    *   [U2] **Bifocal Reading**: 如何阅读 "三区架构" 文档？(Zone 1: 人读 / Zone 3: 机读)。
    *   [U3] **Role Mapping**: 产品、开发、测试、技术负责人分别该从哪一段进入。
    *   [U4] **Command Mastery**: 熟练使用 `/requirement-convergence`, `/quick-dev` 等 Slash Commands。
    *   [U5] **Safety**: 为什么要用 `综合验证（integrated-validator）`？
*   **Action**: 推荐阅读 `README.md`，并尝试先用“阶段理解”再对应到 `/create-spec` 或 `/quick-dev` 的 dry run。

### Track B: The Veteran (老手 / User)
*   **Goal**: "Efficiency"
*   **Modules**:
    1.  **Advanced Skills**: `maglev-design-ux`, `maglev-research`。
    2.  **Shortcuts**: 如何跳过繁琐步骤 (Direct Intent)。
*   **Action**: 介绍最新引入的 Design & Insight 技能。

### Track C: The Architect (Maker)
*   **Goal**: "Evolution"
*   **Modules**:
    1.  **skill-scout**: 如何基于来源池、联网搜索与 AdaptationSpec 生成能力对象。
    2.  **Entry Routing**: 入口路由与对象分发是怎么工作的？
*   **Action**: 挑战任务："尝试用 `skill-scout` 生成一个 Hello World 能力对象。"

## 动作序列

2.  **Challenge**: "接受这个挑战吗？或者你想看别的？"
