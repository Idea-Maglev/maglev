## 必须 vs 推荐小节

| 项目类型 | 必须小节 | 推荐小节 |
|----------|----------|----------|
| Web 应用 | 文档关系、Overview、需求覆盖、UI 状态、组件清单、响应式 | 组件 API、动效、Design Token、可访问性 |
| 移动应用 | 文档关系、Overview、需求覆盖、UI 状态、组件清单 | 手势交互、设备适配、动效 |
| 桌面应用 | 文档关系、Overview、需求覆盖、UI 状态、组件清单 | 快捷键映射、窗口管理 |
| 组件库 | 文档关系、Overview、需求覆盖、组件 API（完整）、Design Token、可访问性 | UI 状态、响应式 |

**不变的必须小节**：文档关系、Overview、需求覆盖表、设计决策。

## 格式规则

| 元素 | 规则 |
|------|------|
| 需求覆盖表 | 必须。引用 I 系 AC，可附加 F 系 AC |
| UI 状态图 | 必须。使用 Mermaid stateDiagram-v2，每个核心场景一张 |
| 组件清单 | 必须。包含「覆盖 AC」列 |
| 组件 API | 推荐。有跨团队协作或组件复用时必须 |
| 响应式策略 | 有多端适配需求时必须 |
| 动效规格 | 有动效需求时必须 |
| Design Token | 有设计系统时必须引用；无则定义关键 token |
| 可访问性方案 | 推荐。有可访问性 AC 时必须 |
