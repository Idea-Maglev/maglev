# Audience Rules — 受众分级规则

本 skill 只持有"受众层级的本质特征 + 容忍度"。**具体某个文件属于哪个层级，由调用方根据本项目语境判断**，skill 不做路径硬编码。

## 三层受众模型

AI 产出的所有文件按读者来源 + 时间维度分三层。

### artifact-session

会话内部产物, 读者就是当前对话双方, 即时消费.

- 本质特征: 读者在场, 共享当前对话上下文; 文件生命周期与会话一致
- 容忍度: **全部会话符号 OK**（如步骤编号、AC 引用、对话指代、迭代标签等）
- 扫描动作: 不必扫描

### artifact-handoff

工程协作产物, 读者是同项目其他工程师 (含未来的自己 6 个月后), 知晓项目协议但不在场.

- 本质特征: 读者具备项目协议知识, 但不参与产生该文件的对话; 文件供同项目内部协作使用
- 容忍度: **工程符号 OK** (协议命名引用 / spec 内部编号在自身 spec 内可用)
- 应剥离: 对话指代 / 时间相对词 / 协作叙事
- 扫描档位: `--severity hard`

### artifact-external

对外产物, 读者完全不在场, 不知道协作上下文.

- 本质特征: 读者跨项目、跨组织或在未来任意时间消费, 既不知道对话也不一定知道项目协议
- 容忍度: **零会话符号**
- 应剥离: 全部 7 类种子规则触发的内容
- 扫描档位: `--severity info` (全部级别)
- **门禁建议**: 有 hard finding 时阻止合并

## 调用约定

调用方在调用本 skill 之前, 必须自行回答两个问题:

1. **该文件的受众层级是什么?** — 由调用方按上述三个本质特征判断。判断依据可以是: 项目自身的目录约定、文件命名约定、调用方已掌握的语境信息、或任何项目级映射规则。
2. **本次扫描使用哪个 severity 档位?** — 按所判定的层级选择: external→`info`、handoff→`hard`、session→不扫。

skill 不内置"路径→层级"的映射表。任何"项目里 `<某路径>` 是 external"这类判断都属于项目责任，不在本 skill 范围。

## 判断疑难时的通用辨析

下列辨析是层级语义本身的边界, 与具体项目无关, 调用方可参考:

- **同一个文件在生命周期内层级会变化**: 一个文件从草稿(session)演进为正式协作件(handoff)再到对外发布件(external), 应在每个阶段重新评估, 不要锁死。
- **意图/动机类文件**: 处于 handoff 与 session 之间。若仅在产生它的小组内消费, 默认 handoff; 若已对外公开, 升为 external。
- **历史依据/归档文件**: 与产生时的受众一致, 通常不必再扫; 但若被引用进入对外文档, 引用处必须按 external 处理。
- **草稿目录的内容**: 草稿期按 session; 一旦其内容进入正式开发流程, 升为 handoff 或 external。
- **不确定时**: 默认 `external` (从严), 让作者主动降级为更宽松的层级。
