# Test Design Surface Workflow

**Goal**: 在质量层统一形成测试设计，而不是让测试规划与测试用例 skill 碎片化平铺。

## 流程 (Process)

1. 识别测试对象
2. 设计覆盖策略
3. 建议测试层次
4. 建议支撑产物

## 进入条件 (Entry Conditions)

- 当前已有 requirements、spec 或实现结果可分析
- 需要先形成统一测试设计，再决定后续验证动作

## 退出条件 (Exit Conditions)

1. 已输出统一测试设计建议
2. 已明确下一步是补测试、生成测试用例还是转交

## 最小产物 (Minimum Deliverables)

- 测试对象识别结果
- 覆盖策略结果
- 测试层次建议
- 支撑产物建议
