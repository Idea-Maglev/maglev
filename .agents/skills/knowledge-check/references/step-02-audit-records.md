# Step 2: Audit Records

## 目标

检查关键知识资产是否已被记录到正确位置。

## 动作

1. 检查 thinking 是否已落盘。
2. 检查 solution / design / requirements 是否已落盘。
3. 检查 references / archive 是否已归位。
4. 检查 `contributors/contribution_log.md` 是否已更新。

## 判定规则

- 先判断是否已有正确落点，再判断是否缺内容。
- 不要求所有材料都进入同一文档，重点是“是否已归到正确位置”。
- 贡献记录只在本轮确有新贡献事实时才要求更新。

## 输出格式

- `record_audit_result`
- `missing_records`
- `blocker_records`

## 输出

- 一份记录完整性判断
- 一组明确缺口
