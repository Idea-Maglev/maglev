# Knowledge Check Workflow

**Goal**: 在高价值探索、会话切换或任务收尾前，执行知识沉淀检查，避免思考资产流失。

## 流程 (Process)

1. 扫描本轮资产
2. 审查记录完整性
3. 校验边界
4. 输出缺口

## 进入条件 (Entry Conditions)

- 本轮产生了值得长期保留的思考、方案或治理判断
- 会话即将切换、结束，或准备进入收尾阶段
- 需要确认知识资产是否还停留在对话中

## 退出条件 (Exit Conditions)

满足以下条件即可结束：

1. 已明确哪些资产已沉淀、哪些未沉淀
2. 已明确这一步是否误滑入现实结晶链
3. 已给出最小补齐动作

## 最小产物 (Minimum Deliverables)

- 资产清单
- 记录完整性结论
- 边界结论
- 缺口清单与补齐建议

## 步骤架构

- **Micro-Steps**: 严格遵循 `step-*.md`
- **Isolation**: 内存中只加载当前步骤
