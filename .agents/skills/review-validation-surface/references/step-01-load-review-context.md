# Step 1: Load Review Context

## 目标

先明确当前正在审什么，以及要对照什么。

## 动作

1. 确认当前结果对象：代码、diff、实现文件或交付结果。
2. 确认当前关联依据：requirements、spec、接口约束或其他前置输入。
3. 明确 review 范围和边界。

## 输出格式

- `review_scope`
- `available_artifacts`
- `reference_basis`
