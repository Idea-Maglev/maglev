# Design

## 1. 需要被区分的四层名字

至少应明确区分：

1. `formal_action_name`
   - 给人理解的正式动作名
2. `skill runtime name`
   - 运行面真实对象名
3. `workflow filename / slash entry`
   - 面向使用的入口名
4. `catalog relation target`
   - 系统内部关系引用名

## 2. 当前治理重点

本主题应至少回答：

1. 四层名字之间允许什么关系
2. 什么情况下必须同轮同步修改
3. 什么时候需要双写兼容
4. 什么样的新命名请求应该被阻断

## 3. 预期输出

1. 一份 naming governance rule
2. 一份最小检查清单
3. 一份允许保留差异的白名单原则
