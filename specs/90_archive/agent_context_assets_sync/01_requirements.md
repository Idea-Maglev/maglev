# Requirements

## 1. 核心目标

让 Maglev 在初始化和升级场景中，具备对 `AGENTS.md` 与 `llms.txt` 的 AI 上下文充分性与漂移风险自检能力。

## 2. 必须满足

1. `init` 时能检查项目是否具备最小 AI 上下文资产
2. `update` 时能检查这两个文件是否已与当前 Maglev 能力结构脱节
3. 能输出明确的缺口判断，而不是笼统说“建议补一下”
4. 不能以原样复制 Maglev 仓库根文件作为默认方案
5. 必须区分：
   - 平台通用要求
   - 项目私域内容

## 3. 需要解决的现实问题

1. skill / workflow 结构升级后，AI 指令层继续停留在旧名字
2. 新项目没有最小可用的 AI 上下文提示
3. 老项目升级后，AI 入口规则与实际能力结构分裂
4. 用户项目中的这两个文件可能混入上游私有内容，导致错误指导

## 4. 当前不做

1. 不直接重做整个 installer 架构
2. 不直接把 `AGENTS.md` / `llms.txt` 原样做成发行制品
3. 不在本轮处理所有用户自定义文档
