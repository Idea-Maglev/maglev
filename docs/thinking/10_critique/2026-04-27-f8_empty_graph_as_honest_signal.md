# F8 空图：当承诺撞上现实

> **Date**: 2026-04-27
> **Status**: active
> **Trigger**: F8 认知地图骨架落地后第一次跑生成器（commit `dbe5209`）

## 1. 事实

P0 主线 F8 落地后第一次跑 `cognitive_map.py --module docs/thinking`：

- 9 个节点（00_meta、10_critique、20_architecture、30_philosophy、40_paper、50_alignment、60_case、70_retrospective、90_archive）
- **0 条边**

注入 root INDEX.md 后，Mermaid 图只剩节点，没有任何连线。

## 2. 为什么是 0 条边

按 AC-F8-4，仅 `status != draft` 入图。当前所有 107 个叶子的 frontmatter 中：

- 绝大多数没有 `status` 字段（按保守默认 = `draft`）
- 即使加上 `--include-drafts` 强制把所有节点入图，跨段 markdown 链接也几乎为零（实测 `grep -E '\.\.\/[0-9]{2}_[a-z_]+/.*\.md'` 全仓只匹配 1 条疑似引用）

**结论**：图为空是事实，不是 bug。

## 3. 这暴露了什么

F8 的承诺是"位段化的认知地图让你像走进宫殿一样导航"。但宫殿要有走廊。当前事实：

1. **房间已建好**（9 个位段全 active，结构完成）
2. **走廊尚未铺设**（叶子之间没有显式的 `linked_to`，也没有跨段引用）
3. **status 字段缺失**（默认 draft 把所有节点排除出图，连"潜在边"都看不到）

承诺与现实之间存在 status 字段补全 + linked_to 声明这一层落差。这不是 F8 的 bug，是 F8 把 D2（叶子 frontmatter 补全）这一层数据债**实证化**了。

## 4. 空图的诚实价值

把空图保留下来，比强行造一个有边的"演示图"更诚实：

- 它把"位段化已完成"和"知识图谱已可用"两件事**清楚地分开**
- 它给后续推 D2 提供了**自带 KPI**（每补一份 frontmatter，图就多一个节点权重；每写一组 `linked_to`，图就多一条边）
- 它让"骨架 = 完成度 X%"和"数据 = 完成度 Y%"两个指标解耦

如果当时偷懒手编一张假图塞进 INDEX，下次跑生成器就要在"覆盖手编"和"保留手编"之间纠结——这是典型的反向腐蚀。

## 5. 推论：诚实信号 > 表面绿色

这一观察可以泛化为一条原则：

> **当机制能够生成空结果时，让它生成。空结果是数据真相的镜像，不要用伪数据掩盖。**

适用范围：

- 文档索引（本案例）
- 测试覆盖率（不要用 mock 假装覆盖）
- 监控仪表盘（不要把无数据画成 0 而非"无数据"）
- AC 全做率（不要把"未做"算作"部分做"以提高百分比）

反面是 status.md 顶部诚实摘要的反义词："基础设施落地，主题未成"——这种诚实摘要本身就是同一原则的产物。

## 6. 后续动作

- **D2 推进**：批量给已沉淀的 active 文档补 `status: active` + `linked_to`，让图慢慢长出
- **不做**：手编假边、手画假图
- **可选**：在 cognitive_map.py 加 `--report-missing-status` 选项，主动暴露未声明 status 的叶子数量
